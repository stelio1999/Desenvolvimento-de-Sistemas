# Cronograma do Projeto - Monitora Wuaze

**Duração Estimada:** 12 Sprints (1 semana cada) = 3 Meses

| Sprint | Período | Objetivo Principal | Atividades Principais | Entregas |
| :--- | :--- | :--- | :--- | :--- |
| **Sprint 1** | Semana 1 | Planeamento e Fundação | Definição do backlog, configuração do ambiente, modelagem da base de dados. | Backlog priorizado, repositórios Git configurados, DER. |
| **Sprint 2** | Semana 2 | Backend e Autenticação | Implementação de APIs de autenticação (registo/login), gestão de parentes e crianças. | API de Autenticação e Gestão de Utilizadores. |
| **Sprint 3** | Semana 3 | Código de Ativação | Implementação da geração de código único (backend) e validação (API). | Módulo de Código de Ativação. |
| **Sprint 4** | Semana 4 | App Base | Configuração do projeto Android, implementação do registo do dispositivo e validação do código no app. | App funcional a validar códigos e registar IMEI. |
| **Sprint 5** | Semana 5 | Monitorização (Localização) | Implementação do serviço de localização em background, recolha e envio para o Firebase. | Módulo de Localização (app + API). |
| **Sprint 6** | Semana 6 | Web Dashboard (Base) | Desenvolvimento do dashboard do parente com listagem de crianças, indicadores e mapa básico. | Dashboard web funcional com localização em tempo real. |
| **Sprint 7** | Semana 7 | Monitorização (SMS e Chamadas) | Implementação da leitura de SMS e histórico de chamadas no app e APIs para receção dos dados. | Módulos de SMS e Chamadas (app + web). |
| **Sprint 8** | Semana 8 | Monitorização (Contactos e Teclado) | Implementação da leitura de contactos e captura de teclado no app. APIs correspondentes. | Módulos de Contactos e Teclado (app + web). |
| **Sprint 9** | Semana 9 | Funcionalidades Avançadas Web | Histórico de localizações, detalhes de SMS/chamadas/contactos no painel web. | Funcionalidades de histórico e detalhes no web. |
| **Sprint 10** | Semana 10 | Módulo Admin e Pagamentos | Desenvolvimento do dashboard do admin, gestão de utilizadores e módulo de pagamentos. | Painel Admin com gestão de pagamentos. |
| **Sprint 11** | Semana 11 | Testes e Otimização | Testes de integração, usabilidade, consumo de bateria, correção de bugs críticos. | Sistema testado e estabilizado. |
| **Sprint 12** | Semana 12 | Lançamento e Documentação | Preparação do ambiente de produção, publicação do APK, documentação do utilizador. | Sistema no ar, APK disponível, documentação final. |
