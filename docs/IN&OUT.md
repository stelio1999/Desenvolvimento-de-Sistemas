Aqui está a definição do **IN & OUT (Escopo do Projeto)** para o Monitora Wuaze:

---

| IN (Dentro do Escopo) | OUT (Fora do Escopo) |
|:---|:---|
| Desenvolvimento da aplicação móvel em Kotlin para Android | Desenvolvimento de aplicação para iOS (iPhone) |
| Funcionamento em background (modo invisível) sem ícone ou notificações | Funcionalidade de bloqueio remoto do dispositivo |
| Monitorização de localização em tempo real | Monitorização de redes sociais (conteúdo de mensagens WhatsApp, Instagram, etc.) |
| Monitorização de SMS, chamadas e contactos | Gravação de chamadas ou ambiente (áudio/vídeo) |
| Monitorização de teclado (textos digitados) | Controlo parental de tempo de uso do dispositivo |
| Dashboard web para visualização dos dados | Acesso público aos dados das crianças |
| Definição de zonas seguras (casa, escola) com alertas | Funcionalidade de chamada ou SOS diretamente do app |
| Gestão de pagamentos mensais em Meticais (MZN) | Integração com sistemas bancários para pagamento automático |
| Autenticação de utilizadores (parentes e admin) | Marketplace ou loja de aplicações |
| Geração e validação de código único por criança | Suporte multilíngue (apenas português) |
| Armazenamento e sincronização offline de dados | Funcionalidade de videochamada |
| Suporte para Android a partir da versão 6.0 (API 23) | Versão web para a criança visualizar os seus próprios dados |
| Uso de OpenStreetMap para mapas gratuitos | Uso de Google Maps (evitando custos de licenciamento) |

---

Esta definição ajuda a evitar **scope creep** (expansão descontrolada do escopo) e mantém o foco no **MVP (Produto Mínimo Viável)** .
